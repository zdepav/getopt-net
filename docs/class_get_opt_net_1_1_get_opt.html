Parses program arguments based on options .

Parameters
argsProgram arguments to process.
optionsList of allowed options separated by commas (,).
Each option can have multiple names, separated by a vertical bar (|). A name is either a single letter or digit (for short options) or a sequence of letters and digits, optionally with words separated by hyphens (for long options). Name can't start or end with a hyphen, no two hyphens can be next to each other. Option name used in parsers and in the returned object is the first one specified.
Option can also have modifiers that are placed after the last name and can be in any order. Allowed modifiers are:
modifier meaning
! required option
: requires value
+ option can be used repeatedly
"requires value" modifier can be optionally followed by a letter, specifying the type of the value. If type is specified and no parser is supplied for this option, a default one is used. Accepted types are:
letter .NET type
s string (default type for option values)
i int
n double
t DateTime (in any format supported by DateTime.TryParse)
f FileInfo (value must be a valid path)
d DirectoryInfo (value must be a valid path)
autoAddHelpAndVersionOptionsIf true, automatically adds –help and –version options if they are not specified in options .
Default: true
ignoreRequiredOnHelpAndVersionIf true and args contains –help or –version options, "required" option modifier is ignored.
Default: true
singleDashLongOptionsIf true, all options are long options and are prefixed with a single hyphen.
If false, short options are prefixed with a single hyphen and can be used in groups, long options are prefixed with two hyphens.
Default: false
valuesAfterOptionsIf true, non-option arguments must appear after the last option argument.
Default: false
includeUnusedOptionsIf true, options that do not appear in args are included in the returned object with null as their value.
If false, options that do not appear in args do not appear in the returned object.
Default: false
parsersA dictionary with functions used to parse option values. Keys are option names, values are functions with two string arguments (option name, value), that return the parsed value.
Returns
A ParseOptions object with options and values extracted from args .
Exceptions
ArgumentNullExceptionThrown when args or options is null.
InvalidOptionsExceptionThrown when args contains invalid options

Remarks

Options string can not contain any whitespace, only ASCII characters are supported.
Options without a value modifier have bool type, with true as their value.
Repeatable options without a value modifier have int type, with number of occurences as their value.

Examples

// Simple short options example
var parsedArgs = GetOpt.Parse(args, &quot;n,e,E&quot;);
// Multiple names and typed value modifier example
var optionsString = &quot;date|d:s,file|f:f,reference|r:f,set|s,utc|universal|u&quot;;
var parsedArgs = GetOpt.Parse(args, optionsString);
// Custom parser example
var parsers = new Dictionary<string, Func<string, string, object?>> {
[&quot;file&quot;] = (_, path) =>
File.Exists(path)
? Path.GetFullPath(path)
: throw new FileNotFoundException($&quot;File \&quot;{path}\&quot; not found.&quot;)
};
var parsedArgs = GetOpt.Parse(args, &quot;file|f:f&quot;, parsers: parsers);
